# ADA Title II Compliance Project - City of Bowie
## Project Context Document

### Executive Summary
The City of Bowie (population >50,000) must comply with ADA Title II regulations by April 24, 2026. They have multiple document repositories, web systems, and third-party platforms that must achieve WCAG 2.1 AA compliance. They have not yet begun gap analysis or compliance planning.

### Critical Deadline
**April 24, 2026** - Federal ADA Title II compliance deadline

### Technical Ecosystem Overview

#### Document Management Systems
1. **Laserfiche Cloud** (Primary DMS)
   - Recently migrated to cloud
   - Stores public and internal documents
   - Requires accessibility governance
   - No built-in accessibility validation

2. **Internal File Servers**
   - Heavily used by staff
   - Not accessible off-network (VPN required)
   - No accessibility controls
   - Migration candidate

3. **OneDrive/SharePoint** (Underutilized)
   - Microsoft 365 environment
   - Todd wants migration here
   - Better governance capabilities
   - Supports accessibility workflows

#### Web Content Management
1. **CivicPlus CMS** (Primary Website)
   - Manages www.cityofbowie.org
   - Has accessibility checker (limited scope)
   - **Critical Limitation**: Only scans CivicPlus modules
   - Does NOT scan third-party links or external systems
   - Managed by Communications, not IT

2. **Third-Party Linked Systems**
   - Munis (Finance system)
   - ActiveNet (Parks & Recreation)
   - Alert Bowie pages
   - Other vendor platforms

#### Existing Tools (Client-Owned)
- Adobe Acrobat Pro (CD-ROM subscription)
- Microsoft Office Suite (accessibility checkers)
- CivicPlus Accessibility Tools
- Laserfiche Cloud
- OneDrive/SharePoint

### Compliance Gap Analysis (Preliminary)

#### Known Gaps
1. **Website Coverage**
   - CivicPlus checker insufficient for full compliance
   - Third-party systems not assessed
   - External links not validated
   - Internal web apps not evaluated

2. **Document Accessibility**
   - No systematic PDF remediation process
   - Existing documents not audited
   - No accessible document templates
   - No creation workflow governance

3. **Staff Training**
   - No accessibility training program
   - Staff unaware of creation standards
   - No departmental champions
   - No ongoing education plan

4. **Vendor Compliance**
